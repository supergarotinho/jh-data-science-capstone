The goal of this project is to try to predict the next word in a sentence. In order to do that we are going to use an text corpus called HC Corpora (www.corpora.heliohost.org). This corpus were collected from different sources: blogs, newspapers, magazines and twitter posts.

We are going to use some Natural Language Processing techniques to process the corpus and extract the statistical knowledge necessary for the model. First, we are going to pre-process and clean the corpus.

What we have done so far:

  1. Made some basic exploration around the corpus files
  2. Load a sub-sample of the corpus into R
  3. Transformations and cleaning process on the data:
    1. Tokenize it into words
    2. Set all letters to lowercase
    3. Remove some profanity words
    4. Remove numbers
    5. Remove ponctuation
  4. Exploring the frequencies of words

Pre-processing and cleaning

  1. Sub-sample the data files.
  2. Remove punctuation and numbers: For our task of predicting the next word, numbers and ponctuation does not seems to improve results.
  3. Set all letters to lowercase
  4. Profanity filtering: Removing profanity words that we do not want to predict.
  5. Tokenization: Identifying appropriate tokens such as words, punctuation, and numbers.

Some needed definitions: ngrams: n-gram is a contiguous sequence of n words in a text. ngram modeling: ngram modeling is the task of extracting the probability of a given ngram (sequence of words) in a corpus. This is usefull for predicting the probability of given word based on the last “n-1” words.
